# @stackpress/lib Docs Wireframes - Round 2

This is the full grayscale wireframe set for review. It is not production site
code and intentionally avoids final color, typography, imagery, and brand
styling.

## Pages

- `index.html` - reference hub homepage
- `events.html` - Events category page
- `system.html` - System category page
- `data.html` - Data category page
- `routing.html` - Routing category page
- `api-reference.html` - representative API article page
- `mobile-nav.html` - explicit mobile drawer state

## Shared Files

- `styles.css` - grayscale wireframe styling
- `script.js` - review-only mobile nav toggle behavior

## Review Questions

- Is this the complete page set needed before creative design?
- Do the four focused category pages have enough structure, or should any route
  directly from the homepage to API articles?
- Does the representative API article cover the right article sections?
- Is one explicit mobile drawer state enough for the wireframe phase?
- Are Queue, Runtime, and Types acceptable as secondary API areas for now?

## Approval Path

If this full page set is approved, the next step is the wireframe handoff and
then creative design direction. If not, revise the named pages or states before
moving forward.
