# @stackpress/lib Docs Creative Draft - Round 1

This is a static creative design-review artifact based on the approved Round 2
wireframes. It is not production site code.

## Direction

The design keeps the approved reference-hub structure and applies a restrained
Stackpress visual system:

- blue Stackpress brand accents
- white and pale-blue reading surfaces
- dark code blocks for developer scanning
- fixed desktop API rail
- calm article pages
- clearer top actions: `GitHub` and `npm`

## Pages

- `index.html` - reference hub homepage
- `events.html` - Events category page
- `system.html` - System category page
- `data.html` - Data category page
- `routing.html` - Routing category page
- `api-reference.html` - representative API article page
- `mobile-nav.html` - explicit mobile drawer state

## Review Questions

- Does the Stackpress blue treatment feel aligned with the attached logo?
- Is the design distinct enough without making article pages noisy?
- Do the card, table, sidebar, and code-block treatments feel consistent?
- Is `npm` the right topbar label, replacing the ambiguous `Package` action?
- Should the creative system stay light-only, or do we need a dark-mode concept
  in a later round?

## Approval Path

If this creative direction is approved, the next step is a refinement round for
any visual adjustments the user names. If no adjustments are needed, the
creative phase can move to handoff before production implementation.
