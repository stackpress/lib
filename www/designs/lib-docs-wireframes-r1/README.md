# @stackpress/lib Docs Wireframes - Round 1

This is a grayscale, low-fidelity design-review artifact. It is not production
site code.

## Scope

- Reference hub homepage for experienced developers
- Focused category page pattern
- API reference article pattern
- Mobile navigation drawer state

## Structural Direction

The homepage prioritizes four primary lanes:

- Events
- System
- Data
- Routing

Queue, Runtime, and Types remain visible as secondary API references.

## Review Questions

- Does the homepage feel like a fast reference hub instead of onboarding?
- Are Events, System, Data, and Routing the right first-screen lanes?
- Is the category page useful enough, or should categories route directly to
  article pages?
- Does the API article order work: import, when-to-use, API, example, related?
- Is the mobile drawer priority correct?

## Next Step

If this structure is approved, the next round should revise any hierarchy issues
and then move into creative design direction using the Stackpress brand.
