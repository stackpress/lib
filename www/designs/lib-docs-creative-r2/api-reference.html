API article

EventEmitter

Article pages stay calm and lookup-oriented: import first, decision rule next, API tables, example, and related links.

Import

import { EventEmitter } from '@stackpress/lib';

When to use it

Use EventEmitter when your event names are exact strings and you want ordered async listeners, hooks, and status results.

Constructor

type Events = {
  saved: [string];
};

const emitter = new EventEmitter<Events>();

Properties

NameTypeNotes
beforeEventHookRuns before each task and can abort.
afterEventHookRuns after each task and can abort.
listenersfrozen copyListener registry keyed by event.

Methods

MethodReturnsNotes
emit()Promise statusExecutes matching listeners in queue order.
on()thisAdds a priority-aware listener.
use()thisMerges listeners from another emitter.

Example

const emitter = new EventEmitter();

emitter.on('saved', async id => {
  console.log('saved', id);
}, 10);

await emitter.emit('saved', 'abc123');
